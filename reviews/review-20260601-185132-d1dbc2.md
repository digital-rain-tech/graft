## Findings

### Medium

1. File path: `src/graft/analysis/jasper_complexity.py:19`  
   Severity: medium  
   Category: Diff  
   Description: `_JAVA_CALLOUT_RE` is likely to miss real Java callout patterns such as `$P{foo}.bar()` or `$F{x}.toString()` because it only matches `new ...(` and `).method(`-style cases. That can undercount `java_callout_count` and misclassify some reports as `automatic` or `assisted` when they should be `manual`.  
   Suggested fix: Define the intended callout surface explicitly and broaden detection accordingly, for example by matching general method-call patterns and then excluding known-safe methods. Add fixtures and tests for the missed expression shapes.

2. File path: `src/graft/analysis/jasper_complexity.py:66`  
   Severity: medium  
   Category: Diff  
   Description: `max_nesting_depth = int(report.metadata.get("max_nesting_depth", 0))` will raise on malformed non-numeric metadata. That makes `analyze` brittle when the report comes from a non-`JasperReader` source or unexpected metadata is injected.  
   Suggested fix: Wrap the cast in `try/except (TypeError, ValueError)` with a safe fallback, or promote `max_nesting_depth` to a typed field on `Report`.

3. File path: `/.github/`  
   Severity: medium  
   Category: Holistic  
   Description: No CI workflows are present, so lint and test checks are not enforced on pushes or pull requests. That weakens regression protection, especially for agent-driven edits.  
   Suggested fix: Add a minimal GitHub Actions workflow that runs `pytest`, `ruff check .`, and optionally `ruff format --check .`.

4. File path: `/`  
   Severity: medium  
   Category: Holistic  
   Description: The repository has a root `CLAUDE.md` but no `AGENTS.md` files in major directories. That leaves agents without local operating guidance for `src/graft/`, `tests/`, and `docs/`, which was part of the requested readiness criteria.  
   Suggested fix: Add `AGENTS.md` files in major directories covering conventions, file ownership, testing patterns, and common pitfalls, then add `CLAUDE.md` symlinks or document the policy for relying on the root file.

5. File path: `SAFETY.md`  
   Severity: medium  
   Category: Holistic  
   Description: The safety documentation promises a full audit trail, including LLM call details, but the current codebase does not show an implemented logging or audit subsystem and translation is still stubbed. This is documentation-to-implementation drift in a safety-sensitive area.  
   Suggested fix: Either implement structured audit logging and document where those records live, or mark the audit-trail claims as planned rather than current behavior.

6. File path: `CLAUDE.md`, `src/graft/cli.py`  
   Severity: medium  
   Category: Holistic  
   Description: The contributor guidance references LLM/provider behavior and CLI capabilities that are not present in the current CLI surface, including unimplemented `translate` and `validate` behavior. That increases onboarding friction and can mislead both humans and agents.  
   Suggested fix: Bring `CLAUDE.md` and related docs in line with the actual CLI, or add explicit placeholder flags and commands that fail with clear not-implemented messaging.
