### 3. Tableau support depends on optional `lxml`, but docs and runtime registration do not guard that dependency

- Line number: `src/graft/readers/registry.py:44`
- Severity: high
- Category: Diff, Holistic
- Description: Tableau reading is enabled by importing and registering `TableauReader`, but `lxml` remains only in the optional `tableau` extra. Also update contributor/dev install docs, add `tableau` to the documented dev install command, include `lxml` in `dev`, or define an `all` extra used by contributors and CI.

## Medium Severity

### 4. CLI export can corrupt machine-readable output by printing through Rich

- File path / directory: `src/graft/cli.py`
- Line number: `165`
- Severity: medium
- Category: Diff
- Description: `export` writes serialized JSON/Markdown through `console.print(result)`, which applies Rich markup parsing and line wrapping. Report content containing Rich tags like `[bold]...[/bold]` can be altered, and long JSON strings can be wrapped with inserted newlines, corrupting machine-readable output.
- Suggested fix: Emit raw serialized exports with `click.echo(result, nl=False)` or use Rich with markup disabled and wrapping controlled, especially for JSON output.
